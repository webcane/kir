Create src/kir/compiler/documents/adapters/markdown_it_adapter.py. This is the ONLY file that imports markdown_it. Module docstring must declare this boundary (following yaml_repository.py's boundary docstring pattern).

Define MarkdownItAdapter class:
- __init__(self) -> None: creates self._md = MarkdownIt() (the MarkdownIt parser instance)
- parse(self, text: str) -> list[Section]: implements MarkdownParserPort.parse() by splitting the Markdown text into Sections at every heading level (H1–H6) per D-01.

The heading-splitting algorithm using markdown-it-py token stream:
1. Call tokens = self._md.parse(text) to get the token list.
2. Walk the token list. When a "heading_open" token is encountered, the immediately following "inline" token's content is the heading text. After the "heading_close", all subsequent non-heading tokens contribute content to the current section until the next "heading_open".
3. Content before the first "heading_open" becomes a preamble section with heading="" (empty string).
4. Build each Section with heading=<inline token content after heading_open> and content=<concatenated inline/paragraph content, stripped>.
5. Discard empty trailing sections (no heading and no content).

The token type sequence from markdown-it-py for "## Hello World\n\nbody text" is:
  heading_open (tag="h2"), inline (content="Hello World"), heading_close,
  paragraph_open, inline (content="body text"), paragraph_close.
Content tokens for body text appear as paragraph_open / inline / paragraph_close groups. Collect inline token content (token.type == "inline") that is NOT immediately inside a heading (i.e., the previous non-fence token is not "heading_open").

Track state with: current_heading (str), current_content_parts (list[str]), and a boolean in_heading to skip inline content inside heading blocks. Append a Section to sections list whenever a new heading is encountered (and the accumulated current section is non-empty) or at EOF.

Return type is list[Section] — a domain type — so ParsePass never needs to import markdown_it. If text is empty or has no structure, return a single preamble Section with heading="" and content=text.strip() (or an empty list if text is blank).

- @register_pass("parse") def parse_pass(ir: Document, ctx: CompilerContext) -> Document:
  - Call sections = ctx.parser.parse(ir.source) — the MarkdownParserPort call; parser is the MarkdownItAdapter in real use, FakeMarkdownParser in tests.
  - Return ir.model_copy(update={"sections": tuple(sections)})
  - No LLM calls. No direct import of MarkdownItAdapter.

Create src/kir/compiler/documents/passes/section.py:
- @register_pass("section", depends_on=("parse",)) def section_pass(ir: Document, ctx: CompilerContext) -> Document:
  - Normalizes each section's content: strip leading/trailing whitespace from Section.content, replace multiple consecutive blank lines with a single blank line.
  - Returns ir.model_copy(update={"sections": tuple(Section(heading=s.heading, content=s.content.strip()) for s in ir.sections)})
  - This is a normalization pass — it does not change the heading or add/remove sections.

Create src/kir/compiler/documents/passes/metadata.py:
- Import hashlib for SHA-256 checksum computation.
- Import Checksum from kir.core.domain.value_objects.
- @register_pass("metadata", depends_on=("parse", "section")) def metadata_pass(ir: Document, ctx: CompilerContext) -> Document:
  - Compute SHA-256 checksum of ir.source: checksum_hex = hashlib.sha256(ir.source.encode()).hexdigest()
  - Derive title from ir.sections: the heading of the first section with a non-empty heading, or "Untitled" if no headings exist.
  - Derive id as a URL-safe slug of the title: lowercase, replace spaces with hyphens, keep alphanumeric and hyphens only. If title is "Untitled", id = "untitled".
  - Return ir.model_copy(update={"checksum": Checksum(value=checksum_hex), "title": title, "id": doc_id, "language": "en"})
  - Source stays unchanged (already in ir.source from initial Document construction).

All pass imports follow: from __future__ import annotations; imports only from kir.core.*; no kir.llm imports.

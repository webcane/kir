<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| prompt template → Agent.run() | Document sections appear in the user-turn position only; the system prompt (instructions=) must be the fixed extract_v1.md template — never interpolate raw document text into the system position |
| pydantic_ai → LLMPort seam | Only llm/pydantic_ai_adapter.py crosses this boundary — all other packages see only LLMPort |
| LLM cache key → CachePort | A degenerate key (empty component) would cause cross-document cache collision |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | LLM prompt injection via Markdown content | mitigate | Document sections must be passed as data to Agent.run(prompt) user-turn only — extract_v1.md is the fixed system instructions; no raw document text appears in the instructions= / system_prompt= position. Enforced at ExtractConceptsPass composition time (Plan 03). |
| T-02-04 | Information Disclosure | API key leakage | mitigate | PydanticAIAdapter model arg is a config string (e.g. "anthropic:claude-sonnet-4-6"); API keys come from environment variables read by the provider library — never hardcoded in pydantic_ai_adapter.py, prompts, or YAML artifacts. LLM-03 tests use FakeLLMAdapter with no credentials. |
| T-02-05 | Tampering | Degenerate LLM cache key collision | mitigate | LLMCacheKey.build() raises ValueError on any empty component — tested in test_cache.py::test_missing_cache_key_component_raises. This is the same security guardrail documented in AI-SPEC.md Section 6. |
| T-02-SC | Tampering | pydantic-ai-slim supply chain | accept | Package Legitimacy Audit in RESEARCH.md confirms Approved status — official Pydantic team product on PyPI. No slopcheck [SUS] or [SLOP] verdict. |
</threat_model>

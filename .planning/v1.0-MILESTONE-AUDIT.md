## Unsatisfied Requirements

### STOR-01 — Artifacts not persisted during compilation

**Requirement:** Each artifact (document, concept, relation, taxonomy node, alias, conflict, metadata) is stored as an individual YAML file — no monolithic JSON.

**Status:** PARTIAL — component proven, integration missing

**Root cause:** Phase 1 verified `YamlFileRepository` works correctly in isolation (one YAML file per artifact, path-traversal safe). But Phase 2's `DocumentCompiler.compile()` never calls `ctx.repository.save()`. The compiled `Document` IR is returned to the caller as an in-memory Python object with no persistence path. No YAML artifact is written to disk.

**Evidence:** Integration checker confirmed `grep -rn "repository.save\|ctx.repository" src/kir/compiler/` returns zero results.

---

### STOR-02 — No output directory created during compilation

**Requirement:** Generated kir/ output is written to a directory separate from raw sources; raw source files are never modified by the compiler.

**Status:** PARTIAL — depends on STOR-01

**Root cause:** STOR-02 requires an output directory to exist during compilation. Since `DocumentCompiler` never calls the repository, no output directory is created or managed. The path isolation logic in `YamlFileRepository` is correct but unreachable from the Phase 2 compilation flow.

4. **IN-03** `tests/core/test_import_boundaries.py:36` — Boundary audit covers `core/domain/` only, not `core/` (passes/ports currently clean but unguarded).
5. **Style × 4** — `from __future__ import annotations` in `registry.py`, `base.py`, `cache_port.py`, `repository_port.py` violates STYLE_GUIDE.md; these are Phase 1 files never touched by Phase 2.
6. **Style coupling** — `tests/core/passes/fakes/fake_llm_port.py` has `from __future__ import annotations` + `TYPE_CHECKING` co-dependency; removing either independently causes `NameError` at runtime.
